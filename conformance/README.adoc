= A2ML Conformance Corpus

// SPDX-License-Identifier: CC-BY-SA-4.0

Local positive and negative fixtures for the A2ML validator, indexed by
link:manifest.a2ml[manifest.a2ml].

== Layout

* `valid/` — fixtures that must validate with zero errors.
* `invalid/` — fixtures that must error under `INPUT_STRICT=true`.

== Rules exercised

Identity (`name` / `project` / `agent-id`), `version`, a license marker in the
first ten lines, and well-formed `[attestation]` / `[section]` blocks. Files
under `.machine_readable/` are identity-exempt but not license-exempt;
`*AI-MANIFEST*` files are identity- and version-exempt.

Negative fixtures are classified as: missing identity (error); and missing
version, missing license header, empty attestation, and malformed heading
(warnings that become errors under strict mode).

== Authoring gotcha

A negative fixture's COMMENT must not contain the literal license token, or a
missing-license case will self-pass.

== Verify

[source,sh]
----
# positive: zero errors
INPUT_PATH=conformance/valid   INPUT_STRICT=true  <validate-a2ml.sh>
# negative: every case errors
INPUT_PATH=conformance/invalid INPUT_STRICT=true  <validate-a2ml.sh>
----
